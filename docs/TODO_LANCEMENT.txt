║ 5. SÉCURITÉ AVANT OPEN SOURCE                                  ║
╚════════════════════════════════════════════════════════════════╝

□ Vérifier qu'aucun secret n'est dans le repo public :
  - .env, .env.local, secrets.json → AUCUN dans le repo
  - service-account.json → JAMAIS commité
  - clés API privées (Mistral, Tavily) → uniquement secrets GitHub Actions

□ Les clés Firebase client (apiKey dans index.html / app.js) sont
  PUBLIQUES par design. La sécurité vient des Firestore Rules.
  → Vérifier les Firestore Rules : Console Firebase → Rules
  → Bonnes règles : un user ne peut lire/écrire que ses propres docs

□ Activer Firebase App Check (anti-abuse, anti-scraping)
  Console Firebase → App Check → reCAPTCHA v3

□ Activer Firebase Security Rules pour Storage si utilisé
